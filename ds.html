DeadStock - Mobile Application

UX DESIGN AND iOS ENGINEERING • FACEBOOK UNIVERSITY ENGINEERING INTERN • SUMMER 2020

GITHUB REPO

Background: During my University Engineering Internship at Facebok, I was tasked with the challenge of designing, engineering, and implementing an industry-level app idea in about 10 weeks. Me, being the sneakerhead that I am, decided to create DeadStock, an application that allows users to see how the sneaker market is doing, track sneaker pricing history, and help investors/buyers alike plan their next purchase.

Inspired by Apple's "stocks" app, DeadStock is a simply designed app meant purely for information consumption rather than a client for purchase. When compared to StockX, DeadStock provides similar sneaker pricing metrics, but without the added buying/selling/bidding features for the sake of ease and simplicity.

Objective: The objective of this project was to design and implement an industry-level sneaker "stocks" app.


Phase 1: Design
Started by laying out usability params and capabilites, laid out screen archetypes, used Procreate to sketch initial wireframe. Progressed to InVision and Figma to create more solid wireframes and mockups to pass on to implementation.

Phase 2: Scraper
Due to the non-existence of a Sneaker API to get data from, turned to creating my own StockX sneaker scraper. Used Selenium and BeautifulSoup to scrape Jordan 1 Data (test subset of ~800 sneakers) and loaded the information into a JSON file.

Phase 3: iOS Engineering
Used Objective-C and Parse backend to implement the application. Involved creating custom Sneaker and User objects, User Authentication, and calls to News API to display sneaker news data. Visit this link for more info on technical implementation.

Screen Archetypes:

  • Log-In Screen: user can log into an existing account
  • Registration Screen: user can register a new profile w/ new login credentials, blocks registration of profile with already existing email or username and prompts user to re-enter
  • User Profile Screen: user can view and edit their profile elements like profile picture, username, password, email, etc.
  • Sneaker Dashboard Screen: user can search for specific sneakers and have them actively displayed on their dashboard
  • Sneaker Search Screen: navigated to upon tapping "add sneaker" feature on the Sneaker Dashboard Screen, pulls up a list of sneakers to select from
  • Sneaker Details Screen: displays organized pricing metrics (52-week pricing, last sale, volatility, colorway, etc.) for a specific sneaker
  • News Screen: user can view top stories from sneaker news sources like solecollector.com and sneakernews.com
  • News Details Screen: upon tapping on an article from the News Screen, the user can view the entire article, author, and publication date
